## システム概要

本システムは、Google Apps Script (GAS) とGoogleスプレッドシートを使用したシフト管理自動化システムです。

### 主な機能
- 従業員による休み希望のWeb提出
- シフト案の自動生成
- 11個のシフトルールチェック
- Googleカレンダーへの自動登録

### システム構成
- **フロントエンド**: GAS Webアプリ（SPA）
- **バックエンド**: Google Apps Script
- **データベース**: Googleスプレッドシート
- **カレンダー**: Googleカレンダー（6グループ分）

---

## 前提条件

### 必要なもの

#### 1. Googleアカウント
- **管理者用アカウント**: 1つ（システム管理者用）
- **要件**: Google Workspace アカウント推奨（無料のGmailアカウントでも可）

#### 2. 必要な権限
- Google Apps Script へのアクセス権限
- Google スプレッドシートの作成権限
- Google カレンダーの作成・共有権限

#### 3. 準備する情報
- 職員リスト（氏名、役職、グループ、雇用形態など）
- 各職員のログインID・パスワード
- グループ構成（1～6のグループ分け）

#### 4. 推奨環境
- PC（Windows/Mac）
- Google Chrome または Microsoft Edge ブラウザ
- インターネット接続

## 導入手順

### ステップ1: Googleカレンダーの準備

グループごとにカレンダーを作成します。

#### 1-1. カレンダーを作成（6個）

1. [Googleカレンダー](https://calendar.google.com/) を開く
2. 左側メニューの「他のカレンダー」の「+」をクリック
3. 「新しいカレンダーを作成」を選択
4. 以下の情報を入力：
   - **名前**: `シフト - グループ1`
   - **説明**: `グループ1のシフト`
   - **タイムゾーン**: `(GMT+09:00) 日本標準時 - 東京`
5. 「カレンダーを作成」をクリック
6. グループ2～6についても同様に作成

#### 1-2. カレンダーIDを取得

各カレンダーについて以下の手順でカレンダーIDを取得します：

1. Googleカレンダーで作成したカレンダー名をクリック
2. 「設定と共有」をクリック
3. 下にスクロールして「カレンダーの統合」セクションを表示
4. **「カレンダーID」** をコピー（例: `abc123@group.calendar.google.com`）
5. メモ帳などに6つ全てのカレンダーIDを記録

```
グループ1: abc123@group.calendar.google.com
グループ2: def456@group.calendar.google.com
グループ3: ghi789@group.calendar.google.com
グループ4: jkl012@group.calendar.google.com
グループ5: mno345@group.calendar.google.com
グループ6: pqr678@group.calendar.google.com
```

#### 1-3. カレンダーを共有（任意）

他のスタッフに閲覧権限を付与する場合：

1. カレンダーの「設定と共有」を開く
2. 「特定のユーザーと共有」セクションで「ユーザーを追加」
3. メールアドレスを入力
4. 権限を選択（「予定の表示（全詳細）」推奨）
5. 「送信」をクリック

### ステップ2: Googleスプレッドシートの作成

#### 2-1. 新規スプレッドシートを作成

1. [Google Drive](https://drive.google.com/) を開く
2. 「+ 新規」→「Googleスプレッドシート」→「空白のスプレッドシート」
3. タイトルを「**シフト管理システム - データベース**」に変更
4. URLをコピーして保存（後で使用）

#### 2-2. スプレッドシートIDを確認

URLから以下の部分をコピー：
```
https://docs.google.com/spreadsheets/d/【ここがスプレッドシートID】/edit
```

例：
```
https://docs.google.com/spreadsheets/d/1a2b3c4d5e6f7g8h9i0/edit
→ スプレッドシートID: 1a2b3c4d5e6f7g8h9i0
```

---

### ステップ3: Google Apps Scriptプロジェクトの作成

#### 3-1. GASプロジェクトを作成

1. [Google Apps Script](https://script.google.com/) を開く
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「**シフト管理システム**」に変更
4. 「無題のプロジェクト」の部分をクリックして名前を変更

#### 3-2. ソースコードをコピー

GitHubリポジトリから以下のファイルをコピーします：

**【重要】以下の順番でファイルを追加してください**

##### ① Config.gs を追加

1. GASエディタで「ファイル」の「+」をクリック
2. 「スクリプト」を選択
3. ファイル名を `Config` に変更
4. GitHubの `Config.gs` の内容を全てコピー＆ペースト
5. Ctrl+S (Mac: Cmd+S) で保存

##### ② DataService.gs を追加

同様の手順で `DataService.gs` を追加

##### ③ AuthService.gs を追加

同様の手順で `AuthService.gs` を追加

##### ④ ShiftService.gs を追加

同様の手順で `ShiftService.gs` を追加

##### ⑤ CalendarService.gs を追加

同様の手順で `CalendarService.gs` を追加

##### ⑥ Code.gs を更新

1. 最初から存在する `Code.gs` をクリック
2. 既存の内容を全て削除
3. GitHubの `Code.gs` の内容を全てコピー＆ペースト
4. 保存

##### ⑦ index.html を追加

1. 「ファイル」の「+」をクリック
2. 「HTML」を選択
3. ファイル名を `index` に変更
4. GitHubの `index.html` の内容を全てコピー＆ペースト
5. 保存

#### 3-3. スプレッドシートとの連携設定（重要）

**方法1: Code.gs から実行（推奨）**

1. GASエディタで `Code.gs` を開く
2. 上部の関数選択で `getSpreadsheetId` を選択
3. 「実行」ボタンをクリック
4. 初回実行時に承認が求められるので承認
5. これで自動的にスプレッドシートIDが設定されます

**方法2: スクリプトプロパティに手動設定**

1. GASエディタで「プロジェクトの設定」（歯車アイコン）をクリック
2. 「スクリプト プロパティ」セクションの「スクリプト プロパティを追加」
3. 以下を入力：
   - **プロパティ**: `SPREADSHEET_ID`
   - **値**: ステップ2でコピーしたスプレッドシートID
4. 「スクリプト プロパティを保存」をクリック

#### 3-4. 権限の承認

1. GASエディタで `Code.gs` を開く
2. 関数選択で `initializeAllSheets` を選択
3. 「実行」をクリック
4. 「承認が必要です」というダイアログが表示されたら「権限を確認」
5. Googleアカウントを選択
6. 「詳細」→「シフト管理システム（安全ではないページ）に移動」
7. 「許可」をクリック

**承認する権限**:
- Google スプレッドシートの表示、編集、作成、削除
- Google カレンダーの表示と編集
- 外部サービスへの接続

### ステップ4: スプレッドシートの初期化

#### 4-1. スプレッドシートとGASを連携

1. ステップ2で作成したスプレッドシートを開く
2. メニューから「拡張機能」→「Apps Script」をクリック
3. 新しいGASプロジェクトが開く
4. このプロジェクトは使用しないので閉じてOK

**または**、GASエディタから直接初期化：

1. GASエディタで `Code.gs` を開く
2. 関数選択で `initializeAllSheets` を選択
3. 「実行」をクリック
4. 実行ログに「初期化完了」と表示されればOK

#### 4-2. 初期化の確認

スプレッドシートに以下のシートが作成されていることを確認：

- ✅ `M_職員` - 職員マスタ
- ✅ `M_シフト` - シフトマスタ（デフォルトシフト付き）
- ✅ `T_シフト休み希望` - 休み希望データ
- ✅ `T_確定シフト` - 確定シフトデータ
- ✅ `M_設定` - 設定マスタ

#### 4-3. シフトマスタの確認

`M_シフト` シートに以下のデータが登録されていることを確認：

| シフトID | シフト名 | 開始時間 | 終了時間 | 備考 |
|---------|---------|---------|---------|-----|
| SHIFT_HAYADE | 早出 | 07:00 | 16:00 | |
| SHIFT_NIKKIN | 日勤 | 09:00 | 18:00 | |
| SHIFT_OSODE | 遅出 | 11:00 | 20:00 | |
| SHIFT_YAKIN | 夜勤 | 17:00 | 09:00 | 翌朝まで |
| SHIFT_YASUMI | 休み | | | |

### ステップ5: Webアプリのデプロイ

#### 5-1. デプロイ設定

1. GASエディタ右上の「デプロイ」→「新しいデプロイ」をクリック
2. 「種類の選択」で「⚙️ Webアプリ」を選択
3. 以下のように設定：

```
説明: シフト管理システム v1.0
次のユーザーとして実行: 自分（管理者のメールアドレス）
アクセスできるユーザー: 全員
```

4. 「デプロイ」をクリック
5. 「ウェブアプリのURL」が表示されるのでコピー

```
例: https://script.google.com/macros/s/AKfycbyXXXXXXXXXXX/exec
```

6. このURLをメモ帳に保存（従業員に配布するURL）

#### 5-2. デプロイの確認

1. コピーしたURLをブラウザで開く
2. ログイン画面が表示されればOK
3. まだログインはできません（職員データ未登録のため）

## 初期データ登録

### ステップ6: 管理者アカウントの登録

まず、システム管理者のアカウントを登録します。

#### 6-1. スプレッドシートで直接登録

1. スプレッドシートの `M_職員` シートを開く
2. 2行目に以下のデータを入力：

| 列 | 項目 | 入力例 | 説明 |
|----|------|--------|------|
| A | 職員ID | STAFF_001 | 一意のID |
| B | 所属 | 本部 | 所属部署 |
| C | 役職 | **管理者** | ⚠️ 必ず「管理者」を含む役職名 |
| D | グループ | 1 | 1～6 |
| E | カレンダーID | (ステップ1で取得したグループ1のID) | カレンダーID |
| F | ユニット | A棟 | 所属ユニット |
| G | 氏名 | 山田太郎 | 氏名 |
| H | 雇用形態 | 常勤 | 常勤/パート |
| I | 喀痰吸引資格者 | TRUE | TRUE/FALSE |
| J | 勤務配慮 | FALSE | TRUE/FALSE |
| K | 有効 | **TRUE** | ⚠️ 必須 |
| L | ログインID | admin | ログイン用ID |
| M | パスワード | admin123 | ログイン用パスワード |

**【重要】注意事項**:
- `役職` に「管理者」「施設長」「主任」「リーダー」のいずれかを含めると管理者権限が付与されます
- `有効` は必ず `TRUE` に設定
- `ログインID` は重複不可
- `パスワード` は平文で保存されるため、初回ログイン後に変更を推奨

#### 6-2. 管理者アカウントでログイン

1. Webアプリを開く
2. 以下でログイン：
   - **ログインID**: `admin`
   - **パスワード**: `admin123`
3. ログインできればOK

### ステップ7: 全職員データの登録

#### 7-1. Webアプリから登録（推奨）

1. Webアプリに管理者アカウントでログイン
2. 上部メニューの「職員マスタ管理」をクリック
3. 「+ 新規追加」ボタンをクリック
4. 職員情報を入力：
   - 職員ID: `STAFF_002`（自動生成される）
   - 氏名: 職員名を入力
   - 役職: 役職を入力
   - グループ: 1～6を選択
   - 雇用形態: 常勤/パートを選択
   - ログインID: ユニークなIDを入力
   - パスワード: 初期パスワードを入力
   - カレンダーID: ステップ1で取得したグループのカレンダーIDを入力
   - 喀痰吸引資格者: 該当する場合はチェック
   - 勤務配慮: 該当する場合はチェック
   - 有効: チェック
5. 「保存」をクリック
6. 全職員について繰り返す

#### 7-2. スプレッドシートから一括登録

多数の職員を登録する場合は、スプレッドシートに直接入力する方が効率的です：

1. Excelで職員リストを作成
2. `M_職員` シートにコピー＆ペースト
3. データ形式に注意：
   - グループ: 数値（1～6）
   - 喀痰吸引資格者: TRUE/FALSE
   - 勤務配慮: TRUE/FALSE
   - 有効: TRUE/FALSE

**テンプレート例（CSV形式）**:
```csv
職員ID,所属,役職,グループ,カレンダーID,ユニット,氏名,雇用形態,喀痰吸引資格者,勤務配慮,有効,ログインID,パスワード
STAFF_002,介護部,介護士,1,abc123@group.calendar.google.com,A棟,田中花子,常勤,TRUE,FALSE,TRUE,tanaka,pass123
STAFF_003,介護部,介護士,2,def456@group.calendar.google.com,B棟,佐藤次郎,パート,FALSE,TRUE,TRUE,sato,pass456
```

### ステップ8: 提出期間設定

従業員が休み希望を提出できる月を設定します。

#### 8-1. 設定値の登録

1. スプレッドシートの `M_設定` シートを開く
2. 以下の行を追加：

**例: 2025年1月の休み希望を受付**

| 設定ID | 設定値 |
|--------|--------|
| SUBMISSION_LOCK_202501 | FALSE |

- `SUBMISSION_LOCK_YYYYMM` の形式で設定ID を作成
- `FALSE`: 提出受付中
- `TRUE`: 締め切り（閲覧のみ）

#### 8-2. 複数月の設定例

```
設定ID                    設定値
SUBMISSION_LOCK_202501   FALSE   (2025年1月: 受付中)
SUBMISSION_LOCK_202502   FALSE   (2025年2月: 受付中)
SUBMISSION_LOCK_202412   TRUE    (2024年12月: 締め切り済み)
```

## 動作確認

### ステップ9: 機能テスト

#### 9-1. ログイン機能のテスト

1. Webアプリを開く
2. 一般職員のアカウントでログイン
3. ログイン成功を確認
4. 管理者アカウントでログイン
5. 管理者メニューが表示されることを確認

#### 9-2. 休み希望提出のテスト

1. 一般職員アカウントでログイン
2. 「休み希望提出」画面を開く
3. 対象月を選択（設定で許可した月）
4. カレンダーから複数の日付をクリック
5. 特記事項を入力
6. 「保存」をクリック
7. スプレッドシートの `T_シフト休み希望` シートにデータが保存されていることを確認

#### 9-3. 職員マスタ管理のテスト

1. 管理者アカウントでログイン
2. 「職員マスタ管理」を開く
3. 登録した職員が一覧表示されることを確認
4. 「編集」ボタンで職員情報を編集
5. 変更が保存されることを確認

#### 9-4. シフト作成機能のテスト

1. スプレッドシートを開く
2. メニューから「📅 シフト管理」→「✨ シフト案作成」
3. 対象年月を入力（例: `2025/01`）
4. 「OK」をクリック
5. `シフト作業用` シートが作成され、シフト案が生成されることを確認
6. 休み希望が反映されていることを確認

#### 9-5. ルールチェック機能のテスト

1. `シフト作業用` シートで一部のシフトを手修正
2. わざと違反を作成（例: 6連勤、遅出→翌日早出など）
3. メニューから「📅 シフト管理」→「✅ ルールチェック」
4. 対象年月を入力
5. 違反箇所が赤くハイライトされることを確認
6. ダイアログに違反内容が表示されることを確認

#### 9-6. カレンダー登録のテスト

1. シフト作業用シートでシフトを調整
2. ルールチェックでエラーがない状態にする
3. メニューから「📅 シフト管理」→「📝 シフト登録」
4. 対象年月を入力
5. 「OK」をクリック
6. Googleカレンダーを開く
7. 各グループのカレンダーにシフトが登録されていることを確認
8. イベントのタイトル、日時が正しいことを確認

## トラブルシューティング

### 問題1: GASの実行でエラーが出る

**症状**:
- 「承認が必要です」と表示される
- 「権限がありません」と表示される

**解決方法**:
1. GASエディタで `Code.gs` を開く
2. 任意の関数を選択して「実行」
3. 権限承認画面で「許可」をクリック
4. 再度実行

### 問題2: スプレッドシートが見つからない

**症状**:
- 「スプレッドシートが見つかりません」というエラー

**解決方法**:
1. ステップ3-3を再確認
2. スクリプトプロパティに正しいスプレッドシートIDが設定されているか確認
3. スプレッドシートのURLからIDを再取得

### 問題3: Webアプリが開かない

**症状**:
- URLにアクセスしてもエラー画面が表示される

**解決方法**:
1. デプロイ設定を確認（ステップ5）
2. 「アクセスできるユーザー」が「全員」になっているか確認
3. 新しいデプロイを作成して、新しいURLを取得

### 問題4: ログインできない

**症状**:
- 「ログインIDまたはパスワードが正しくありません」と表示される

**解決方法**:
1. `M_職員` シートでログインID、パスワードを確認
2. 「有効」フラグが `TRUE` になっているか確認
3. スペースや大文字小文字に注意

### 問題5: カレンダーに登録されない

**症状**:
- シフト登録を実行してもカレンダーにイベントが作成されない

**解決方法**:
1. `M_職員` シートの「カレンダーID」が正しいか確認
2. カレンダーIDにスペースや改行が含まれていないか確認
3. GASにカレンダーへのアクセス権限があるか確認
4. GASの実行ログを確認（「表示」→「ログ」）

### 問題6: カスタムメニューが表示されない

**症状**:
- スプレッドシートに「📅 シフト管理」メニューが表示されない

**解決方法**:
1. スプレッドシートを一度閉じて再度開く
2. GASエディタで `Code.gs` の `onOpen` 関数を実行
3. ブラウザのキャッシュをクリア

## サポート・問い合わせ

導入に関して問題が発生した場合：

1. **GASの実行ログを確認**
   - GASエディタ → 「表示」→「ログ」
   - エラーメッセージを確認

2. **スプレッドシートのデータを確認**
   - データ形式が正しいか
   - 必須項目が入力されているか

3. **ブラウザの開発者ツールを確認**
   - Webアプリで F12 キーを押す
